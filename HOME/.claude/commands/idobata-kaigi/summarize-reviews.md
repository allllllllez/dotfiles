# 井戸端会議 Review Aggregation

あなたは、複数のAIエージェントが独立して実行したコードレビュー結果を集約し、統合レポートを作成する役割を担います。

## タスク

1. `.idobata-kaigi/`ディレクトリ内の、本日の日付（YYYYMMDD形式）を含むすべてのレビューファイルを読み込む
   - パターン: `.idobata-kaigi/$(date +%Y%m%d)-p*-review.md`

2. 各エージェントのレビュー内容を分析し、以下の観点で統合：
   - **重複する指摘**: 複数のエージェントが同じ問題を指摘している場合、重要度が高い
   - **専門分野別の分類**: セキュリティ、パフォーマンス、コード品質、アーキテクチャ
   - **重大度の評価**: Critical > High > Medium > Low
   - **合意度**: 何人のエージェントが同様の指摘をしているか

3. 統合レポートを以下の構成で作成：

```markdown
# 井戸端会議 統合レビューレポート

生成日時: $(date)

## エグゼクティブサマリー

- 参加エージェント数: N
- 指摘事項総数: M
- Critical/High優先度の問題数: X

## 優先度順の推奨事項

### 🔴 Critical（即座に対応が必要）

1. [問題タイトル]
   - **カテゴリ**: セキュリティ/パフォーマンス/品質/アーキテクチャ
   - **合意度**: N/4 エージェント
   - **場所**: ファイル名:行番号
   - **詳細**: 問題の説明
   - **推奨事項**: 修正方法
   - **レビュアー**: どのエージェント（ペルソナ）が指摘したか

### 🟠 High（優先的に対応すべき）

...

### 🟡 Medium（時間があれば対応）

...

### 🟢 Low（将来的な改善案）

...

## カテゴリ別サマリー

### セキュリティ
- 指摘数: X
- 主要な懸念: ...

### パフォーマンス
- 指摘数: Y
- 主要な懸念: ...

### コード品質
- 指摘数: Z
- 主要な懸念: ...

### アーキテクチャ
- 指摘数: W
- 主要な懸念: ...

## ポジティブフィードバック

- 良い点1
- 良い点2

## 結論

総合的な評価と、次のアクションアイテム
```

4. 統合レポートを以下のファイルに保存：

```
.idobata-kaigi/$(date +%Y%m%d)-all-reviews.md
```

## 注意事項

- 各エージェントの独立性を尊重し、専門分野の視点を明記すること
- 単なる羅列ではなく、問題の関連性を考慮して構造化すること
- Critical/Highの問題には、具体的なアクションアイテムを提示すること
- 複数のエージェントが一致した見解は、より信頼性が高いと見なすこと
- バランスの取れたレビューのため、ポジティブな側面も必ず含めること
